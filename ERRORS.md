# Обработка ошибок

В проекте предусмотрена обработка основных ошибок, которые могут возникнуть при запуске программы и во время синхронизации файлов.

## 1. Ошибки конфигурации

Ошибки конфигурации возникают до начала основной работы программы.
В таких случаях программа завершает работу и выводит пользователю понятное сообщение.

### Примеры ошибок конфигурации

#### Отсутствует файл `config.ini`

Если рядом с программой нет файла `config.ini`, пользователь увидит сообщение:

Ошибка конфигурации: Файл config.ini не найден

#### Отсутствует секция `[settings]`

Если в файле `config.ini` нет секции `[settings]`, программа завершится с сообщением:


Ошибка конфигурации: Секция [settings] не найдена в config.ini


#### Отсутствует обязательный параметр

Если в `config.ini` не указан один из обязательных параметров, например `token`, программа завершится с сообщением:


Ошибка конфигурации: Отсутствует обязательный параметр: token


#### Не существует локальная папка

Если путь к локальной папке указан неверно или папка не создана, программа завершится с сообщением:


Ошибка конфигурации: Локальная папка не найдена: ./local_files


#### Некорректный токен

Если токен Яндекс Диска неправильный, программа завершится с сообщением:


Ошибка конфигурации: Неверный токен Яндекс Диска


#### Не существует удалённая папка на Яндекс Диске

Если папка, указанная в `remote_folder`, не создана на Яндекс Диске, программа завершится с сообщением:


Ошибка конфигурации: Удалённая папка не найдена


---

## 2. Ошибки во время работы программы

Ошибки, которые происходят уже после запуска во время синхронизации, не должны останавливать программу.
Они записываются в лог, после чего программа продолжает работу.

### Примеры ошибок во время синхронизации

#### Ошибка чтения локального файла

Например, если файл временно недоступен для чтения, в лог записывается ошибка:


[ERROR] Ошибка чтения файла test.txt: [Errno 13] Permission denied


#### Ошибка загрузки файла на Яндекс Диск

Если во время загрузки файла пропал интернет или API недоступно, ошибка записывается в лог:


[ERROR] Ошибка загрузки файла report.txt: Не удалось загрузить файл. Код: 500


#### Ошибка обновления файла

Если при перезаписи файла произошёл сбой, программа пишет в лог:


[ERROR] Ошибка обновления файла report.txt: Не удалось загрузить файл. Код: 503


#### Ошибка удаления файла

Если файл не удалось удалить с Яндекс Диска, это тоже фиксируется в логе:


[ERROR] Ошибка удаления файла report.txt: Не удалось удалить файл. Код: 404


#### Ошибка получения списка файлов с Яндекс Диска

Если во время очередной проверки не удалось получить список файлов из удалённой папки, в лог записывается сообщение:


[ERROR] Ошибка получения списка удалённых файлов: Не удалось получить список файлов. Код: 403


---

## 3. Общий принцип обработки ошибок

В проекте используется следующий подход:

- ошибки конфигурации считаются критичными;
- при критичных ошибках программа завершает работу;
- ошибки во время синхронизации считаются рабочими;
- рабочие ошибки записываются в лог и не останавливают программу.

---

## 4. Примеры в коде

### Пример обработки ошибки конфигурации


def validate_local_folder(path):
    if not os.path.isdir(path):
        raise ValueError(f"Локальная папка не найдена: {path}")


Если папка не существует, вызывается `ValueError`, и программа завершает работу с понятным сообщением.

### Пример обработки ошибки при запуске программы


try:
    config = load_config("config.ini")
    settings = get_settings(config)
    validate_local_folder(settings["local_folder"])
    validate_remote_access(disk_client)
except ValueError as error:
    print(f"Ошибка конфигурации: {error}")


Здесь ошибки конфигурации перехватываются и выводятся пользователю.

### Пример обработки ошибки во время синхронизации


def _upload_new_files(self, filenames):
    for filename in filenames:
        path = os.path.join(self.local_folder, filename)
        try:
            self.disk_client.load(path)
            logging.info("Загружен новый файл: %s", filename)
        except Exception as error:
            logging.error("Ошибка загрузки файла %s: %s", filename, error)


Если файл не удалось загрузить, программа не завершается, а просто записывает ошибку в лог.

### Пример обработки ошибки при первой синхронизации


def _first_sync(self, local_snapshot):
    try:
        remote_info = self.disk_client.get_info()
    except Exception as error:
        logging.error("Ошибка получения списка удалённых файлов: %s", error)
        return False


Если при первой синхронизации не удалось получить список файлов с Яндекс Диска, ошибка записывается в лог, и первая синхронизация не считается завершённой.

---

## 5. Итог

Таким образом, в проекте разделены два типа ошибок.

### Критичные ошибки

Приводят к остановке программы:

- нет `config.ini`;
- нет секции `[settings]`;
- не указаны обязательные параметры;
- неверный токен;
- не найдена локальная папка;
- не найдена удалённая папка.

### Рабочие ошибки

Не останавливают программу:

- ошибки чтения файлов;
- ошибки загрузки файлов;
- ошибки обновления файлов;
- ошибки удаления файлов;
- временные проблемы сети или API.

Это позволяет сделать работу программы более устойчивой и удобной для пользователя.
