# Сервис синхронизации файлов с Яндекс Диском

## Описание проекта

Данный проект представляет собой консольное Python-приложение для синхронизации файлов между локальной папкой на компьютере пользователя и папкой на Яндекс Диске.

Программа работает в фоновом режиме и через заданный промежуток времени проверяет состояние локальной папки. Если в папке появляются новые файлы, изменяются существующие или удаляются файлы, программа выполняет соответствующие действия на Яндекс Диске:

- новый файл загружает;
- изменённый файл перезаписывает;
- удалённый файл удаляет.

Проект выполнен в рамках учебного задания Skillbox.

---

## Основные возможности

- чтение настроек из файла `config.ini`;
- проверка корректности параметров запуска;
- первая синхронизация при старте программы;
- периодическая проверка локальной папки;
- загрузка новых файлов на Яндекс Диск;
- обновление изменённых файлов на Яндекс Диске;
- удаление файлов с Яндекс Диска, если они были удалены локально;
- логирование действий и ошибок в файл.


### Назначение файлов

- `main.py` - точка входа в программу, чтение конфигурации, запуск цикла синхронизации;
- `sync_service.py` - логика сравнения локальных и удалённых файлов;
- `yandex_disk.py` - класс для работы с API Яндекс Диска;
- `config.ini` - рабочий конфигурационный файл;
- `config.ini.example` - пример шаблона настроек;
- `README.md` - основная документация по проекту;
- `ERRORS.md` - подробное описание обработки ошибок;
- `local_files/` - локальная папка, которая синхронизируется;
- `logs/` - папка с логами программы.

## Требования

Для запуска проекта необходимо:

- Python 3.10+;
- установленная библиотека `requests`.

---

## Установка и запуск

### 1. Установить библиотеку requests

pip install requests

---

### 2. Подготовить локальную папку

Нужно создать папку `local_files`, если её ещё нет.

Пример:
mkdir local_files

---

### 3. Создать папку на Яндекс Диске

В Яндекс Диске нужно заранее вручную создать папку, которая указана в параметре:

remote_folder = skillbox_sync

Например, создать папку skillbox_sync

---

### 4. Заполнить файл config.ini

Пример конфигурации в файле `config.ini.example`

Где:
- `local_folder` - путь к локальной папке;
- `remote_folder` - имя папки на Яндекс Диске;
- `token` - OAuth-токен Яндекс Диска;
- `sync_period` - период проверки папки в секундах;
- `log_file` - путь к файлу логов.

---

### 5. Запустить программу

python main.py

---

## Как работает программа

После запуска программа:

1. считывает настройки из `config.ini`;
2. проверяет наличие локальной папки;
3. проверяет доступ к Яндекс Диску;
4. записывает информацию о запуске в лог;
5. выполняет первую синхронизацию;
6. переходит в бесконечный цикл проверки папки через заданный интервал времени.

---

## Принцип синхронизации

### Первая синхронизация

При первом запуске программа сравнивает содержимое локальной папки и удалённой папки на Яндекс Диске:

- лишние файлы на Яндекс Диске удаляются;
- файлы из локальной папки загружаются или перезаписываются.

### Последующие проверки

На следующих шагах программа сравнивает текущее состояние локальной папки с предыдущим снимком:

- если появился новый файл - загружает его;
- если файл изменился - обновляет его;
- если файл удалён - удаляет его с Яндекс Диска.

## Пример работы программы

### Пример запуска

2026-04-10 16:05:44,860 [INFO] Запуск синхронизации. Локальная папка: ./local_files

2026-04-10 16:05:47,534 [INFO] Первая синхронизация: файл синхронизирован: Инструкция.txt

2026-04-10 16:05:47,534 [INFO] Первая синхронизация успешно завершена

[INFO] Загружен новый файл: test1.txt

### Пример изменения файла

Если изменить содержимое файла `test1.txt`, то после следующей проверки программа запишет:

[INFO] Обновлён файл: test1.txt

### Пример удаления файла

Если удалить файл `test1.txt` из локальной папки, то после следующей проверки программа запишет:

[INFO] Удалён файл: test1.txt

## Логирование

Программа ведёт лог в файл, указанный в параметре `log_file`.

Пример:

./logs/sync.log

В лог записываются:

- запуск программы;
- успешные действия синхронизации;
- ошибки чтения файлов;
- ошибки сетевого взаимодействия;
- остановка программы пользователем.

---

## Остановка программы

Для остановки программы нужно нажать:

Ctrl + C

После этого программа завершится корректно и выведет сообщение:

Программа остановлена пользователем

---

## Обработка ошибок

Кратко:

- ошибки конфигурации приводят к завершению программы с понятным сообщением;
- ошибки во время синхронизации записываются в лог и не останавливают программу.

Подробное описание обработки ошибок и примеры приведены в файле `ERRORS.md`.
